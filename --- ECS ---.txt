--- ECS ---
abstract class Entity{
	
	Component list acting on it rn
	Data common to every entity (id?...)
}
abstract class Component{
	String name;
	int id;
}
class System{
	called by components to change certain things (ex adding gravity to z momentum)
}

class Game{
	loop that creates entities and draws them
	adds components to entities based on key presses
	
}

	Entity archetype creates entity with certain combination of components

	Components are data on entites

	Systems are logic that change the components

in general : Entity relies on Component relies on System
